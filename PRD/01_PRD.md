## 1. 제품 개요

### 한 줄 요약
인스타그램·유튜브·블로그 링크를 붙여 넣으면 장소 정보를 자동으로 저장하고, 저장 이유·출처와 함께 지도에서 다시 탐색할 수 있는 일상형 장소 저장 서비스

### 해결하는 문제
- **정보 분산**: 인스타그램, 유튜브, 블로그, 지도 앱에 저장 장소가 흩어짐
- **맥락 손실**: 시간이 지나면 왜 저장했는지, 어디서 봤는지 기억하기 어려움
- **지도 기반 탐색 부재**: 저장 목록은 있지만 "지도에서 주변 저장 장소 보기"가 안 됨
- **실행 연결 부족**: 저장은 많이 하지만 실제 방문 계획으로 이어지지 않음

### 핵심 가치
| 가치 | 설명 |
|------|------|
| **통합** | 여러 플랫폼 링크를 하나의 지도에서 관리 |
| **맥락 보존** | 출처 URL·저장 이유를 장소와 함께 기록 |
| **지도 기반 재탐색** | 목록이 아닌 위치 중심으로 다시 확인 |
| **실행 연결** | 원문 이동·길찾기로 실제 방문으로 연결 |

## 2. 사용자

### 주요 사용자
- **누구**: 인스타그램·유튜브·블로그에서 맛집·카페·전시·여행지를 즐겨 저장하는 20~30대
- **상황**: 콘텐츠를 보다가 "나중에 가봐야지" 싶을 때, 또는 여행 전 저장한 장소를 정리할 때
- **목표**: 발견한 장소를 빠르게 저장하고, 나중에 지도에서 쉽게 다시 찾기

### 개발자 관점 목표
포트폴리오용 — Java 백엔드 역량 증명
- 도메인 설계, URL 처리 및 플랫폼별 분기 로직, 외부 지도 API 연동, 지도 조회 API 설계, 예외 처리 및 테스트 코드

### 사용자 시나리오
1. 사용자가 인스타그램에서 카페 피드를 보다가 앱을 열고
2. 해당 게시물 URL을 붙여넣으면
3. 자동으로 카페 이름·주소·출처 링크가 저장되고 지도 핀으로 표시된다
4. 다음에 그 지역을 방문할 때 지도를 열어 핀을 확인하고 카카오맵으로 길찾기를 한다

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 링크 자동 추출 | URL 입력 → 플랫폼 구분 → OG 태그 수집 → LLM(gpt-4o-mini)으로 장소 후보 추출 + 콘텐츠 분류 → Kakao 장소 매칭 | P1 (MVP) | 보통 |
| 지도 핀 표시 | 저장 장소를 카카오맵 위에 핀으로 표시 | P1 (MVP) | 보통 |
| 카테고리 관리 | 사용자 정의 카테고리 생성·편집·삭제, 핀 색상 설정 | P1 (MVP) | 간단 |
| 카테고리 필터 | 지도에서 카테고리별 핀 필터링 | P1 (MVP) | 간단 |
| 상세 화면 | 저장 이유·출처 링크·visitStatus 표시 | P1 (MVP) | 간단 |
| 로그인/회원가입 | 카카오 OAuth + 이메일/비밀번호 | P1 (MVP) | 보통 |
| 길찾기 연결 | 카카오맵·네이버지도·티맵 딥링크 | P2 | 간단 |
| visitStatus 변경 | WANT ↔ VISITED 변경 | P2 | 간단 |
| 지도 범위 조회 | 현재 지도 뷰포트 기반 핀 조회 | P2 | 보통 |
| 메모 후속 입력 | 저장 이유를 나중에 추가/수정 | P2 | 간단 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름 — 링크 저장
```
링크 복사 → 앱 열기 → URL 붙여넣기
→ OG 태그 수집 → LLM 장소 후보 추출 (SINGLE_PLACE / MULTI_PLACE / REVIEW_REQUIRED)
→ Kakao 장소 매칭 → 후보 확인 → 카테고리 선택 → 저장 완료
→ 지도에 핀 표시
```

### 핵심 흐름 — 장소 탐색
```
지도 열기 → 핀 확인 → 핀 클릭
→ 상세화면 (저장 이유·출처·visitStatus)
→ (선택) 원문 읽기 or 길찾기
```

## 5. 성공 기준

- [ ] URL 붙여넣기 5초 이내에 장소 핀이 지도에 표시됨
- [ ] 인스타그램·유튜브·블로그 3개 플랫폼 링크 처리 성공률 80% 이상
- [ ] 저장한 장소를 카테고리/지도 범위로 필터링해서 찾을 수 있음
- [ ] 상세화면에서 원본 링크 및 저장 이유를 확인할 수 있음
- [ ] 카카오맵·네이버지도·티맵으로 길찾기 연결이 동작함

---

## 6. 안 만드는 것 (Out of Scope — Phase 1)

- 콜렉션 공유 기능 — 이유: 소셜 기능은 Phase 3 이후
- 주변 장소 탐색(반경 검색) — 이유: 지도 API 고급 기능, Phase 3
- 오프라인 캐시 — 이유: 구현 복잡도 대비 MVP 효과 낮음
- 사진 업로드 — 이유: 출처 썸네일로 대체 가능
- 팔로우·소셜 피드 — 이유: 개인용 도구로 먼저 증명
