# ADR-004: 링크 정보 추출 전략 — OG 태그 파싱 + 카카오 장소 API 매칭

- 날짜: 2026-03-18
- 상태: 결정됨

---

## 배경

"URL을 붙여넣으면 자동으로 장소가 저장된다"는 핵심 기능을 구현해야 한다.
URL에서 장소 이름·주소·좌표를 어떻게 추출할 것인가?

인스타그램, 유튜브, 블로그는 각각 다른 구조를 가지며,
공식 API 없이 콘텐츠에서 장소 정보를 뽑아내야 한다.

---

## 결정사항

**2단계 파이프라인**을 채택한다:

```
1단계: URL → OG 태그 파싱 (Jsoup, 타임아웃 5초)
         → title, thumbnail, description 추출

2단계: title → 카카오 장소 검색 API (타임아웃 5초)
         → name, address, latitude, longitude 매칭
```

- OG 태그 제목을 카카오 장소 검색 쿼리로 사용
- 카카오 매칭 실패 시 `kakaoMatchFound = false` 반환 → 프론트에서 수동 입력 UI 표시
- 플랫폼 구분: URL 패턴 매칭 (instagram.com / youtube.com / blog.naver.com 등)

---

## 대안

### A. 플랫폼별 공식 API 사용
- 인스타그램 Graph API, 유튜브 Data API
- 장점: 정확한 콘텐츠 메타데이터
- 단점: 인스타그램 API 접근 제한 심각 (비즈니스 계정 필요), API 키 관리 부담 증가, MVP 범위 초과

### B. Headless Browser (Puppeteer/Playwright) 스크래핑
- 장점: JS 렌더링 후 동적 콘텐츠 파싱 가능
- 단점: 서버 리소스 과다 소비, 실행 속도 느림, 배포 복잡도 증가

### C. 장소명 직접 입력 (자동 추출 없음)
- 장점: 구현 단순
- 단점: "링크 붙여넣으면 자동 저장" 핵심 가치 훼손

### D. AI(GPT 등)로 URL 콘텐츠 파싱
- 장점: 정확도 높음, 플랫폼 무관
- 단점: API 비용 발생, 응답 지연, MVP 의존성 증가

---

## 결과

### 긍정적
- 공식 API 없이도 대부분의 플랫폼에서 OG 태그 파싱 가능
- 카카오 장소 API로 정확한 좌표·주소 획득
- 5초 타임아웃으로 응답 지연 상한 제어

### 부정적
- 인스타그램은 로그인 없이 OG 태그 접근이 제한될 수 있음
- OG 태그 제목이 장소명이 아닌 경우 카카오 매칭 정확도 하락
- 카카오 API 요청 한도(월 30만 건 무료) 초과 시 비용 발생

### 중립적
- 매칭 실패 케이스는 수동 입력 UI로 보완 (04_PROJECT_SPEC.md ALWAYS DO 항목)
- 성공률 목표: 3개 플랫폼에서 80% 이상 (PRD 성공 기준)
- 향후 플랫폼별 파서를 별도 전략 클래스로 분리 가능 (Strategy Pattern)
